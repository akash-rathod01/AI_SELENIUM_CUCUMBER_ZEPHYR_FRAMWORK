 # AI Selenium Cucumber BDD Zephyr Framework

## Overview
Python automation framework that combines Selenium WebDriver, Behave BDD, and a curated utility toolkit. The repository currently ships with the Saucedemo end-to-end checkout scenario and is ready for additional projects.

## Key Features
- Behave feature files with page-object style step helpers
- Selenium driver bootstrap for Chrome (default) and Firefox (via `BROWSER=firefox`)
- Centralised reporting (`reports/<project>` HTML summary and raw Behave JSON)
- Optional Allure integration (`allure-results/<project>`) when the CLI is installed
- AI maintenance suite for snapshots, flaky-test detection, locator healing insights, broken-link diagnostics, and automated flake triage (`--ai-maintenance`)
- Reusable utilities for data loading, self-healing, visual checks, AI-driven test ideas, blueprint crawling, scenario planning, and automation scaffolding

## Project Structure
```
features/                   # Shared Behave assets (if any)
projects/saucedemo/         # Example project bound to saucedemo.com
  features/                 # Feature files, step definitions, environment hooks
pages/                      # Page objects or shared page helpers
tests/test_runner.py        # Command-line entry point for Behave execution
reports/<project>/          # Generated summaries, zipped artifacts
allure-results/<project>/   # Allure JSON (requires Allure CLI to visualise)
utils/                      # Support modules (data loader, NLP generator, planners, etc.)
tools/                      # CLI helpers (crawler, planner, scaffolder)
data/blueprints/            # Stored site blueprints generated by the crawler
```

## Prerequisites
- Python 3.11+
- Google Chrome (and driver) and/or Mozilla Firefox (and geckodriver)
- Optional: Allure CLI for HTML reports (`allure --version` should succeed). Run `powershell -File tools/install_allure_cli.ps1` to download the latest release locally and add `tools\allure\bin` to your `PATH`.

## Installation
```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Copy `.env.example` to `.env` if project-specific environment variables are required (e.g., `GRID_URL`, credentials).

## Running Tests
Execute the unified runner and specify the project key:
```bash
python tests/test_runner.py --project saucedemo
```

Enable the AI maintenance workflow to produce a resilience snapshot and health report:
```powershell
python tests/test_runner.py --project saucedemo --ai-maintenance --maintenance-base-url https://dotesthere.com
```
Reports are written to `reports/maintenance/` and snapshots to `snapshots/`.

Run the maintenance suite without executing tests:
```powershell
python tools/ai_maintenance.py --project saucedemo --base-url https://dotesthere.com
```

Generate a standalone flake triage report against an existing Behave JSON output:
```powershell
python tools/ai_flake_triage.py reports/auto_completion_projcet/behave-report.json --print-json
```

To force Firefox instead of Chrome:
```bash
$env:BROWSER = "firefox"
python tests/test_runner.py --project saucedemo
Remove-Item Env:BROWSER
```

### Remote/Grid Execution
Set `GRID_URL` in the environment or `.env`; the driver factory automatically switches to `webdriver.Remote`.

## Reporting
- `reports/saucedemo/summary.html`: quick pass/error counts and metadata
- `reports/saucedemo/behave-report.json`: raw Behave output for downstream consumption
- `reports/saucedemo/latest_report_bundle.zip`: zipped artifacts for sharing
- `allure-results/saucedemo/`: ingest with `allure serve allure-results/saucedemo` after installing the Allure CLI
- `reports/allure_dashboard.html`: landing page with links to the latest Allure dashboards for every registered project (generated via `python tools/allure_dashboard.py`)

Regenerate every project's dashboard after a test run:
```powershell
python tools/allure_dashboard.py
```

## Extending The Suite
1. Duplicate the project folder structure under `projects/<new_project>`.
2. Register the project in `config/project_registry.py` with its feature path and optional environment entries.
3. Model additional pages in `pages/` or within the project folder.
4. Leverage utilities in `utils/` to add self-healing, visual snapshots, or AI-generated tests.

### Generating Page Blueprints
Use the crawler CLI to capture a site blueprint (links, forms, metadata) that downstream planners can consume:
```powershell
python tools/crawl.py --url https://www.saucedemo.com --project saucedemo --max-depth 1
```
Blueprint JSON lands under `data/blueprints/<project>/` with a timestamped filename and a `latest.json` pointer for quick retrieval. Pass `--stdout` to inspect the payload without writing to disk.

Increase crawl depth and include additional domains if the target renders content dynamically:
```powershell
python tools/crawl.py --url https://www.saucedemo.com --project saucedemo --max-depth 2 --include-domains saucedemo.com saucedemoassets.com
```

### Planning Scenarios
Translate a blueprint into structured scenario ideas ready for NLP generation:
```powershell
python tools/plan_scenarios.py --project saucedemo
```
Use `--blueprint <path>` to target a specific snapshot and `--output plans.json` to persist the ideas.

### Scaffolding Automation Assets
Take the ideas and generate Behave feature files and step stubs:
```powershell
python tools/scaffold_from_blueprint.py --project saucedemo --feature-name auto_blueprint
```
Add `--use-llm` if you have the NLP generator configured to call an external model. The scaffolder writes the outputs to `projects/<project>/features/` and `projects/<project>/features/steps/`.
